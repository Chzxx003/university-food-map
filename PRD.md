# PRD — 大学城美食地图

**文档版本**: v1.0
**创建日期**: 2026-03-13

---

## 一、背景与目标

### 1.1 背景

深圳大学城（清华大学深圳研究院、北京大学深圳研究院、南方科技大学、哈工大深圳校区、深圳大学）周边聚集了大量餐饮商户，但学生群体普遍面临"吃什么"的决策困境：

- 餐饮信息分散在美团、大众点评、小红书等多个平台
- 缺乏专门面向学生群体的本地化美食指南
- 现有平台内容不够精准，广告推广干扰信息获取

### 1.2 产品目标

搭建一个**大学城美食地图**网页应用，帮助学生高效发现、浏览、选择大学城周边餐厅，解决"吃什么"的决策痛点。

---

## 二、用户分析

### 2.1 目标用户

**主要用户：大学生**
- 年龄：18~25 岁
- 消费特征：价格敏感，性价比优先，人均 15~40 元为主
- 使用场景：日常一人食、朋友聚餐、情侣约会、深夜宵夜

**次要用户：校园附近居民、教职工**

### 2.2 用户痛点

| 痛点 | 描述 |
|------|------|
| 信息分散 | 需要跨多平台查找，决策成本高 |
| 选择困难 | 不知道吃什么，尤其多人用餐时更难统一 |
| 踩雷风险 | 不了解近期口碑，容易遇到翻车商家 |
| 预算不透明 | 人均价格区间不明确，容易超出预算 |
| 场景匹配差 | 无法按"一人食""聚餐""宵夜"等场景快速筛选 |

## 三、功能需求

### 3.1 功能优先级总览

| 优先级 | 功能模块 |
|--------|----------|
| P0（必做） | 餐厅列表、地图展示、搜索与筛选、餐厅详情页 |
| P1（重要） | 学生场景标签、排行榜、评价与评分 |
| P2（加分） | 随机推荐、收藏夹、按学校区域分区 |

---

### 3.2 P0 功能详细说明

#### 3.2.1 餐厅列表

- 以卡片形式展示餐厅，每张卡片包含：
  - 餐厅名称、封面图
  - 菜系标签（川菜、日料、快餐等）
  - 人均价格
  - 综合评分（1~5 星）
  - 距离（基于所选学校定位）
  - 场景标签（一人食 / 聚餐 / 宵夜 / 约会 / 学生套餐）
- 支持列表视图与网格视图切换

#### 3.2.2 地图展示

- 在地图上以 Pin 标注餐厅位置
- 点击 Pin 弹出餐厅简要信息卡片
- 支持以所选学校为中心缩放地图

#### 3.2.3 搜索与筛选

**搜索**：按餐厅名称、菜系、菜品名称关键词搜索

**筛选条件**：

| 维度 | 选项 |
|------|------|
| 菜系 | 中餐、日料、韩料、西餐、快餐、小吃、甜品… |
| 人均价格 | 0~15 元 / 15~30 元 / 30~50 元 / 50 元以上 |
| 场景 | 一人食 / 聚餐 / 宵夜 / 约会 / 学生套餐 |
| 评分 | 4 星以上 / 4.5 星以上 |
| 营业状态 | 仅看营业中 |

#### 3.2.4 餐厅详情页

- 餐厅基本信息：名称、地址、电话、营业时间、人均
- 图片轮播
- 菜品推荐（图片 + 名称 + 价格）
- 用户评价列表（评分 + 文字 + 时间）
- 场景标签
- 地图位置预览

---

### 3.3 P1 功能详细说明

#### 3.3.1 学生场景标签

为每家餐厅打上一个或多个场景标签，帮助学生快速匹配需求：

- `一人食` `聚餐` `宵夜` `情侣约会` `学生套餐` `外卖友好` `自习续杯`

#### 3.3.2 排行榜

- 本周热门榜（按访问量）
- 高分好评榜（按综合评分）
- 高性价比榜（按评分/人均比）

#### 3.3.3 评价与评分

- 用户可提交：综合评分（1~5 星）+ 文字评价（20~200 字）
- 评价展示最新 10 条
- 评价支持点赞

---

### 3.4 P2 功能详细说明

#### 3.4.1 今天吃什么（随机推荐）

- 用户点击「帮我选」按钮，系统随机推荐 1 家符合当前筛选条件的餐厅
- 解决多人用餐时的选择困难问题

#### 3.4.2 收藏夹

- 用户可将餐厅加入收藏，下次直接访问
- 收藏数据存储在本地（localStorage）

#### 3.4.3 按学校分区

- 首页顶部可切换所在大学（A 大学 / B 大学 / C 大学…）
- 地图与列表随之更新，优先展示该校周边餐厅

## 六、页面流程

```
用户进入首页
  → 选择所在学校
  → 浏览地图 / 列表
  → 使用搜索 / 筛选缩小范围
  → 点击餐厅卡片进入详情页
  → 查看菜品、评价、位置
  → 收藏 / 发起评价
```

## 七、设计原则

1. **学生感** — 视觉风格活泼年轻，避免商务冷淡风
2. **信息密度适中** — 卡片信息一目了然，不过度堆砌
3. **手机优先** — 主要使用场景为移动端碎片化浏览
4. **快速决策** — 所有核心操作（搜索、筛选、随机推荐）3 步以内完成
